The problem
The tools exist. The people who need them can't use them.
Cybersecurity intelligence is real-time and freely available — CISA, NVD, VirusTotal, PhishTank. None of it is accessible to non-technical users.
  • VirusTotal returns raw engine flag counts — meaningless without context
  • NVD requires CPE product string knowledge to query
  • Base LLMs have a training cutoff — blind to current threats
  • No single tool retrieves, synthesizes, and explains in plain English

Phishing agent · Component deep dive
Two parallel evidence streams. Verdict by priority rules, not probability.
Stream 1 — Live API
  • Submit URL to VirusTotal API v3
  • Receive engine flag counts from 70+ scanners
  • ≥5 malicious → MALICIOUS
  • 2–4 malicious or 3+ suspicious → UNCERTAIN
  • 0 flags + no email signals → SAFE
Stream 2 — Semantic Retrieval
  • Embed URL → query PhishTank namespace
  • Retrieve top-5 similar verified phishing patterns
  • Similarity score >0.35 → activates UNCERTAIN fallback if VT unavailable
Email signal extraction
  • Urgency language patterns
  • Credential harvesting signals
  • Sender spoofing indicators
  • Contributes to verdict when no URL present

Vulnerability agent · Component deep dive
Two-pass retrieval surfaces actively exploited CVEs before theoretical ones.
Pass 1 — Exploited-first
Filter Pinecone cve-vulnerabilities by known_exploited=True. Retrieve only CISA KEV records — confirmed real-world attacks.
Pass 2 — Full semantic search
Unfiltered semantic search across full namespace. Deduplicate against Pass 1 results. Catches HIGH/CRITICAL NVD records.

Anti-hallucination design
  • Only facts from the provided evidence list may be cited
  • CVE IDs not in evidence list are replaced with [redacted]
  • Regex post-generation scan on all CVE identifiers
  • GPT-4o mini synthesis at temperature 0.1
  • No specific tool names or commands in action steps — generic best-practice language only
Contextual intervention selects from 5 templates: malicious not clicked · malicious already clicked · vulnerable · uncertain · safe

Evaluation results · Phishing agent — live VirusTotal
100% precision. 15% recall. The gap is a test design problem, not a system failure.

Why recall is 15%: The 20 phishing test URLs were constructed for evaluation. VirusTotal only flags URLs that have been submitted and scanned by the community. A URL never seen by VT receives zero flags regardless of its structure. This is a test-set limitation — not a system failure.
What is valid: Precision of 100% is real — every MALICIOUS verdict was correct. False positive rate of 0% is real — no legitimate URL was wrongly flagged. The system demonstrates calibrated uncertainty: returning UNCERTAIN rather than fabricating MALICIOUS.
12 / 16
Implementation · Key challenges resolved
Three architectural problems discovered in build. All resolved before evaluation.
Event loop conflict (Python 3.13 + FastAPI + asyncio)
Nesting asyncio.run() inside a running event loop raises RuntimeError. Resolution: execute VirusTotal call in ThreadPoolExecutor with a fresh event loop per thread. nest_asyncio discarded due to TypeError with uvicorn's loop_factory.
NVD product metadata gap — CPE filter failed
NVD records in Pinecone carry no product/vendor metadata fields. The CPE product pre-filter returned zero results for all queries. Resolution: two-pass architecture — Pass 1 uses known_exploited metadata against KEV namespace; Pass 2 is pure semantic search. KEV records carry full product metadata.
PhishTank brand-tag recall metric — invalid evaluation design
93% of PhishTank records carry an "Other" brand tag. Brand-tag Recall@5 produced systematic false negatives — correct phishing retrievals scored as misses. Resolution: evaluation redesigned to measure end-to-end verdict accuracy (precision, recall, FPR) rather than retrieval brand-tag accuracy.
13 / 16
Limitations · Documented before presentation
Four structural limitations. Three with clear remediation paths.
NVD product metadata
Cannot filter by product name before semantic rank. Requires re-indexing 90K records with CPE metadata from NVD raw data (available but not stored).
VirusTotal free-tier rate limit
500 req/day, 4/min. Production requires paid tier or Redis cache keyed by URL SHA-256 hash. Academic program provides 1,000 req/day.
Static Pinecone index
Snapshot as of April 2026. New CVEs, new phishing URLs, new KEV entries not reflected. Requires daily diff sync mechanism — Tier 1 future improvement.
Test set design
Constructed phishing URLs limit live-VT recall validity. Stronger evaluation requires verified historical PhishTank URLs with confirmed VT community exposure.
14 / 16
Future work · Three tiers
Near-term fixes. Production hardening. Industry scale.
Tier 1 — Near-term
  • Daily KEV + PhishTank sync via CISA and PhishTank feeds — upsert only new records
  • Resume NVD download with checkpoint logic; re-index with CPE product metadata
  • Redis URL cache keyed by SHA-256 hash to reduce VT quota consumption
Tier 2 — Production
  • Zero-day detection via WHOIS domain-age lookup (domains < 7 days → UNCERTAIN)
  • URL redirect chain resolution — prevent URL shortener evasion
  • OS-aware intervention: platform-specific update instructions from browser User-Agent
Tier 3 — Scale
  • MISP, AlienVault OTX, Shodan federated integration
  • Browser extension with 500ms latency target and domain reputation cache
  • RLHF feedback loop → quarterly FLAN-T5 fine-tuning
  • Multi-language: detect via LLM, output in Polish/Portuguese/English
